## ?? ТЕСТОВІ СЦЕНАРІЇ CRUD

### 1. Drug (Препарати)
- ? **Create**: Додати новий препарат з категорією та виробником
- ? **Read**: Переглянути список/деталі препаратів
- ? **Update**: Редагувати інформацію про препарат
- ? **Delete**: Видалити препарат (тільки якщо не на складі/в продажах)

### 2. Pharmacy (Аптеки)
- ? **Create**: Додати нову аптеку з адресою
- ? **Read**: Переглянути список/деталі аптек
- ? **Update**: Редагувати дані аптеки
- ? **Delete**: Видалити аптеку (тільки якщо порожня)

### 3. Customer (Клієнти)
- ? **Create**: Додати клієнта (ПІБ, email, телефон)
- ? **Read**: Переглянути список клієнтів
- ? **Update**: Редагувати дані клієнта
- ? **Delete**: Видалити клієнта (без покупок)

### 4. InventoryItem (Запаси на складі)
- ? **Create**: Додати запас препарату в аптеку
- ? **Read**: Переглянути інвентар аптеки
- ? **Update**: Оновити кількість/ціну
- ? **Delete**: Видалити запис (тільки якщо Q=0)

### 5. Sale (Продажі)
- ? **Create**: Створити продаж (з транзакцією)
- ? **Read**: Переглянути список продажів
- ? **Update**: Редагувати дані продажу
- ? **Delete**: Видалити продаж (повернення на склад)

### 6. SaleLine (Позиції продажу)
- ? **Create**: Додати позицію до продажу
- ? **Read**: Переглянути позиції
- ? **Update**: Редагувати кількість/ціну
- ? **Delete**: Видалити позицію

### 7. PurchaseOrder (Закупівлі)
- ? **Create**: Створити замовлення у постачальника
- ? **Read**: Переглянути замовлення
- ? **Update**: Змінити статус
- ? **Delete**: Видалити замовлення

### 8. Prescription (Рецепти)
- ? **Create**: Додати рецепт для клієнта
- ? **Read**: Переглянути рецепти
- ? **Update**: Редагувати дані
- ? **Delete**: Видалити рецепт

## ?? ТЕСТУВАННЯ БІЗНЕС-ЛОГІКИ

### ? Операції, що повинні бути заблоковані:

1. **Видалення препарату** - якщо він на складі
   - Очікування: помилка "Препарат присутній на складі"
   
2. **Видалення аптеки** - якщо там є робітники
   - Очікування: помилка "У аптеці є працівники"

3. **Видалення клієнта** - якщо він робив покупки
   - Очікування: помилка "У клієнта є история покупок"

4. **Оновлення ціни** - на 0 або негативне значення
   - Очікування: помилка "Невалідна ціна"

## ?? ТЕСТУВАННЯ ЗАПИТІВ

### Query 1: Рецептурні препарати (Фільтрація)
```
Очікування: Список препаратів де PrescriptionRequired = true
```

### Query 2: Продажі за період (Агрегація)
```
Очікування: Сума продажів по датам за обраний період
```

### Query 3: Низькі залишки (JOIN)
```
Очікування: Препарати де Quantity <= ReorderLevel
```

### Query 4: Оновити ціну (UPDATE)
```
Очікування: Ціна оновлена для всіх записів з умовою
```

### Query 5: Видалити постачальника (DELETE)
```
Очікування: Видалено, якщо немає пов'язаних замовлень
```

### Query 6: Додати продаж (INSERT + UPDATE)
```
Очікування: Продаж створений, склад зменшений (атомарна операція)
```
